Moonflix

Objective

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Endpoints

Movies

REQUEST URL HTTP METHOD REQUEST DATA BODY FORMAT RESPONSE DATA BODY FORMAT
Return a list of ALL movies to the user. /movies GET None A JSON object holding data about all the movies and status code 200 if sucessful.
Return data (description, genre, director, image URL, whether it is featured or not) about a single movie by title to the user. /movies/[title] GET None A JSON object holding data about the movie containing the title and status code 200 if sucessful. Example: { title: 'The Godfather', description: 'The aging patriarch of an organized crime dynasty transfers control of his clandestine empire to his reluctant son.', genre: { name: 'Drama', description: 'Movies focused on realistic characters, settings, and stories that evoke emotion.' }, director: { name: 'Francis Ford Coppola', bio: 'An American film director, producer, and screenwriter.', birth: '1939', death: null }, imagePath: 'https://example.com/godfather.jpg', featured: true }
Return data about a genre (description) by genre name. /movies/genre/[genreName] GET None A JSON object holding data about the genre and status code 200 if sucessful. Example: { "name": "Science Fiction", "description": "Fictional science that explores the possibilities of the future." }
Return data about a director (bio, birth year, death year) by name. /movies/directors/[directorName] GET None A JSON object holding data about the director containing the name and status code 200 if sucessful. Example: { name: 'Francis Ford Coppola', bio: 'An American film director, producer, and screenwriter.', birth: '1939', death: null }

Users

REQUEST URL HTTP METHOD REQUEST DATA BODY FORMAT RESPONSE DATA BODY FORMAT
Register new user. /users POST A JSON Object holding information about the new user. Example: { username: 'user4', password: 'password4', email: 'user4@example.com', birth: '1993-05-03', favoriteMovies: [ name: 'Titanic' ] } A JSON Object holding data about the new user and status code 201 if created. Example: { username: 'user4', password: 'password4', email: 'user4@example.com', birth: '1993-05-03', favoriteMovies: [ name: 'Titanic' ]}
Update a user by id. /users/:id PUT A JSON Object holding information about the user to update. Example: { "name": "Maria Silva", "email": "user4@example.com", "birth": "1993-05-03", "favoriteMovies": [] } A JSON Object holding new data about the updated user and status code 200 if sucessful. Example: { "id": "123456", "name": "Maria Silva", "email": "user4@example.com", "birth": "1993-05-03", "favoriteMovies": [] }
Add a movie to user's list of favorites. /users/:id/:movieTitle POST None Message confirming that the movie was added.
Delete a movie from user's list of favorites. /users/:id/:movieTitle DELETE None Message confirming that the movie was deleted.
Delete user registration. /users/:id DELETE None Messsage confirming that the email was removed.
